## Design Philosophy — NON-NEGOTIABLE
> "Design for the least technical user in the room, not the most."

- Every screen passes the **3-second rule** — user understands without reading anything
- Maximum 3 primary actions visible per screen
- Mobile-first always — field workers use phones, not desktops
- Error messages in plain English — never raw technical errors
- Accessibility by default — WCAG 2.1 AA minimum
- Inline help, tooltips, and contextual guidance on all forms
- No jargon, no abbreviations, no technical terms in UI copy
- Loading states, empty states, error states — always designed, never unstyled
- Onboarding flows must be resumable — user can always continue where they left off
